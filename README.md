## What It Does

This dashboard connects to the **GitHub Copilot Usage Metrics API** (the new GA API, not the legacy one) and provides a comprehensive view of adoption across all organizations in your enterprise.

### Metrics Collected

**From the GitHub API (per user, per day):**

| Metric | Description |
|--------|-------------|
| Code Generation Activity | Number of code suggestions generated by Copilot |
| Code Acceptance Activity | Number of suggestions accepted by the developer |
| Lines of Code | LOC suggested (add/delete) and LOC actually added/deleted |
| User Interactions | Explicit prompts sent to Copilot |
| Feature Usage | Whether the user used Completions, Chat, Agent Mode, or CLI |
| Chat Modes | Breakdown by agent mode, ask mode, edit mode |
| IDE & Language | Which editor and programming language was used |
| CLI Tokens | Prompt and output tokens consumed via Copilot CLI |

**Calculated by the dashboard:**

| Metric | Description |
|--------|-------------|
| Adoption Rate | Active users / total seats (%) |
| DAU / WAU | Daily and weekly active users |
| Acceptance Rate | Acceptances / generations (%) |
| Wasted Seats | Users with a license who haven't used Copilot |
| Cost per Active User | Total license spend / active users |
| User Categories | Power User, Occasional, Inactive, Never Used |

### Dashboard Views

- **Adoption Overview** - KPIs, DAU/WAU trends, acceptance rate over time
- **Users** - Sortable table with all metrics, click any user for daily detail
- **User Detail** - Per-day charts: code activity, LOC, acceptance rate, top languages, full daily breakdown table
- **Features** - Completions vs Chat vs Agent vs CLI usage distribution
- **Languages & Editors** - Top programming languages and IDE distribution
- **Trends** - Week-over-week evolution, daily and weekly aggregates
- **ROI** - Total lines accepted, cost per user, wasted spend analysis
- **AI Reports** - AI-powered adoption analysis (see below)

### AI-Powered Reports

Connect the dashboard to **Azure AI Foundry** (or any OpenAI-compatible endpoint) to generate executive adoption reports on demand. The AI analyzes all your metrics and produces a structured report with:

1. **Executive Summary** - Overall adoption state
2. **Adoption Analysis** - Rates vs industry benchmarks
3. **Top Performers** - Users with best practices, potential mentors
4. **At-Risk Users** - Never Used, Inactive, Low adoption with specific actions
5. **Feature Adoption** - Which features are underutilized
6. **Trends** - Is adoption going up or down? Why?
7. **Recommendations** - Prioritized, actionable steps
8. **ROI Analysis** - Cost optimization, wasted spend

Reports are stored and can be reviewed at any time from the report history.

## Quick Start (Local Dev)

### Prerequisites

- .NET 8 SDK
- Node.js 20+
- A GitHub Copilot Business or Enterprise license
- A Personal Access Token with scopes: `manage_billing:copilot`, `read:org`, `read:enterprise`

### 1. Clone and configure

```bash
git clone https://github.com/Hiunkeru/gh-copilot-dashboard.git
cd gh-copilot-dashboard
```

Create `src/backend/CopilotDashboard.Api/appsettings.Development.json`:

```json
{
  "GitHub": {
    "Enterprise": "your-enterprise-slug",
    "Organizations": ["your-org"],
    "Token": "ghp_xxxxx"
  },
  "AiFoundry": {
    "Endpoint": "https://your-resource.openai.azure.com/",
    "ApiKey": "your-key",
    "DeploymentName": "o4-mini"
  }
}
```

> The `AiFoundry` section is optional. Without it, everything works except the AI Reports tab.

### 2. Run

```bash
# Terminal 1 - Backend
cd src/backend/CopilotDashboard.Api
dotnet run

# Terminal 2 - Frontend
cd src/frontend
npm install --legacy-peer-deps
npm run dev
```

### 3. Open

Go to **http://localhost:3000**

- If a GitHub token is configured, real data is synced on startup
- If not, sample data with 30 simulated users is loaded
- Re-sync anytime: `curl -X POST http://localhost:5182/api/sync/trigger`

## GitHub API

This dashboard uses the **new Copilot Usage Metrics API** (GA February 2026), not the legacy Copilot Metrics API (deprecated April 2026).

Endpoints used:
- `GET /enterprises/{slug}/copilot/metrics/reports/users-28-day/latest` - User-level metrics
- `GET /orgs/{org}/copilot/billing/seats` - Seat assignments (per org)

The API returns NDJSON files via signed download links. The dashboard streams and parses them line by line.

## Multi-Organization Support

For enterprises with multiple organizations, the dashboard:
- Fetches **metrics at enterprise level** (covers all orgs automatically)
- Fetches **seats per organization** (iterates all orgs)
- Auto-discovers organizations or uses an explicit list in config

## Deployment to Azure

Infrastructure is defined in Terraform under `infra/`:

| Resource | Purpose |
|----------|---------|
| App Service (B1) | .NET API backend |
| Static Web App (Free) | Next.js frontend |
| Storage Account | Table Storage for reports |
| Key Vault | GitHub PAT and AI keys |
| Application Insights | Monitoring |

```bash
cd infra
terraform init
terraform apply -var-file=environments/prod.tfvars -var="sql_admin_password=xxx"
```

CI/CD via GitHub Actions: PR builds on `ci.yml`, deploy on merge to `main` via `cd.yml`.

## Configuration Reference

| Setting | Location | Description |
|---------|----------|-------------|
| `GitHub:Enterprise` | appsettings | Enterprise slug from URL |
| `GitHub:Organizations` | appsettings | Array of org slugs (empty = auto-discover) |
| `GitHub:Token` | appsettings / Key Vault | PAT with copilot scopes |
| `GitHub:LicenseCostPerMonth` | appsettings | USD per seat (19 Business, 39 Enterprise) |
| `AiFoundry:Endpoint` | appsettings / Key Vault | Azure OpenAI endpoint URL |
| `AiFoundry:ApiKey` | appsettings / Key Vault | Azure OpenAI API key |
| `AiFoundry:DeploymentName` | appsettings | Model deployment name |
| `NEXT_PUBLIC_DEV_MODE` | .env.local | Set to `true` to bypass auth in dev |
